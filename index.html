DSA Overview

Data Structures and Algorithms (DSA) form the foundation of efficient programming and problem-solving. Mastering DSA helps you write optimized, scalable, and logical solutions for real-world applications.

  • Data Structures – Ways to store and organize data (Array, Stack, Queue, Tree, Graph).
  • Algorithms – Step-by-step procedures to process data (searching, sorting, traversal).
  • Efficiency – Measured with Big-O notation for time and space complexity.

Array

An Array is a linear data structure storing elements in contiguous memory locations.

  • Fast index access → O(1)
  • Insertion/removal at end → O(1)
  • Search by value → O(n)

Applications:

  • Lists, matrices, buffers, building-block for other structures

Stack

A Stack is a LIFO structure — Last In, First Out.

  • Push — add to top
  • Pop — remove from top
  • Peek — view top

Applications:

  • Call stacks, undo history, parsing expressions

Queue

A Queue is a FIFO structure — First In, First Out.

  • Enqueue — add to rear
  • Dequeue — remove from front
  • Peek — view front

Applications:

  • Scheduling, BFS traversal, task queues

Binary Tree

A binary tree is a hierarchical structure where each node has up to two children (left and right).

Traversals:

  • Inorder: Left → Root → Right
  • Preorder: Root → Left → Right
  • Postorder: Left → Right → Root

Applications:

  • Expression trees, binary search trees, file systems

Algorithms

  1. Linear Search
  2. Binary Search
  3. Bubble Sort
  4. Selection Sort
  5. Insertion Sort
  6. Quick Sort
  7. Merge Sort
  8. Breadth-First Search (BFS)
  9. Depth-First Search (DFS)
  10. Dijkstra's Shortest Path

Pseudocode

// Linear Search
LinearSearch(A, key)
  for i = 0 to len(A)-1
    if A[i] == key return i
  return -1

// Binary Search
BinarySearch(A, key)
  low = 0; high = len(A)-1
  while low <= high
    mid = (low+high)//2
    if A[mid]==key return mid
    if A[mid] < key low = mid+1 else high = mid-1
  return -1
